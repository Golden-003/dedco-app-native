01 — Accueil

Le hero immersif

L'accueil ne montre pas un menu. Il montre une émotion. Le hero prend 60% de l'écran — image plein cadre, overlay gradient du bas, titre en Playfair Display. En dessous, un carrousel horizontal de cards produit (swipe). Pas de grille statique — tout scrolle, tout vit.

Accueil — Hero + Carrousel

Le hero change à chaque ouverture. 5-10 visuels en rotation. L'overlay gradient (transparent → rgba(30,24,19,0.8)) garantit la lisibilité du texte blanc sur n'importe quelle image. Les cards produit sont en swipe horizontal — l'utilisateur découvre sans scroller verticalement.

02 — Inspirations

Masonry grid émotionnel

03 — Marketplace

Grid 2 colonnes + filtres bottom sheet

La marketplace est dense mais respirante. Grid 2 colonnes, cards avec ratio image 4:3. Les filtres s'ouvrent en bottom sheet qui glisse depuis le bas — style Airbnb. L'utilisateur ne quitte jamais la liste.

Bottom sheet — Filtres

Le bottom sheet a un handle (petite barre grise) en haut — l'utilisateur sait qu'il peut le glisser vers le bas pour fermer. Le bouton "Voir N résultats" est sticky en bas du sheet. Les chips de filtre utilisent le même composant que les filtres horizontaux de la marketplace.

04 — Fiche Produit

Image plein écran + bottom sheet info

L'image prend tout l'écran. Les infos produit sont en bottom sheet qui couvre les 40% inférieurs. L'utilisateur peut glisser le sheet vers le haut pour voir plus de détails, ou le rabaisser pour voir l'image. Swipe horizontal = photos suivantes. Pinch = zoom.

05 — Brief

Swipe entre étapes

Le brief n'est pas un formulaire. C'est une conversation visuelle. L'utilisateur swipe entre étapes. Chaque étape montre de grandes icônes tactiles, pas des inputs. La progression est visible en haut — points colorés qui se remplissent.

Brief — Étape 2: Style

La barre de progression en haut utilise des dots (✓ vert = fait, amber = actuel, gris = à venir) reliés par des lignes. Les cartes de style ont une bordure 2px — amber si sélectionnée, border si non. Chaque carte a une icône colorée unique (pas de doublons). Les boutons en bas sont sticky : Retour (ghost) + Continuer (primary, 2x plus large).

06 — Panier & Checkout

Mobile Money, 3 étapes, bottom sheet

Le panier est simple : liste d'articles avec steppers +/-. Suppression par swipe gauche. Le checkout s'ouvre en bottom sheet — 3 étapes : livraison, paiement Mobile Money, confirmation. L'utilisateur ne quitte jamais l'écran.

Checkout — Mobile Money

Le panier montre le total en .font-numeric (chiffres tabulaires alignés). Le stepper +/− utilise le amber pour le "+" (action positive) et le warm pour le "−" (action neutre). 07 — Chat

Coordination contextuelle

Le chat n'est jamais accessible directement. Il s'ouvre depuis le suivi de projet. Le header montre le projet associé (titre, statut, progression). Les bulles sont amber (moi) et warm (artisan). Les actions contextuelles (valider modification, confirmer livraison) sont des boutons inline dans le chat.

Chat — Coordination projet

Le header du chat est foncé (ink) pour se distinguer du contenu. Il affiche le projet (titre, ID, statut, barre de progression). Les actions contextuelles (modification, validation) apparaissent inline dans le flux de chat — pas dans un menu séparé. L'utilisateur peut accepter ou refuser sans quitter la conversation. Le champ de message est un pill arrondi avec bouton d'envoi amber circulaire.

08 — Bottom Navigation

5 onglets, icônes amber

La bottom nav est fixe, blanche, avec une légère ombre au-dessus. L'onglet actif a son icône et son label en amber-dark. Les autres sont en ink-mute. Pas de badge sur les onglets — les badges vivent dans le header (panier, notifications).
